üì¶ YaiJS Import Examples

YaiJS offers flexible import strategies to match your project needs. Choose from bundles for convenience or granular imports for optimal tree-shaking.

‚ÑπÔ∏è About YEH & Bundles

YEH (Yai Event Hub) is a standalone library published separately as @yaijs/yeh. It's the foundation for YaiJS but can be used independently in any project.

Bundles are CDN-only: The bundle files (yai-bundle.js, etc.) are designed for CDN usage and include YEH automatically. For NPM installations, install both @yaijs/yeh and @yaijs/core separately.

ESM-only: YaiJS is built with modern ES modules. Browser usage requires <script type="module"> - standard script tags won't work. Supports all modern browsers (Chrome 61+, Firefox 60+, Safari 11+, Edge 79+).

üì¶ NPM Package Imports

NPM Install (Recommended) Recommended

Install both YEH and YaiJS packages, then import what you need. Perfect for production builds with tree-shaking. 

// Install both packages
// npm install @yaijs/yeh @yaijs/core

// Import YEH foundation
import { YEH } from '@yaijs/yeh';

// Import YaiJS components (granular)
import { YaiTabs } from '@yaijs/core/tabs';
import { YaiTabsSwipe } from '@yaijs/core/swype';
import { YaiViewport } from '@yaijs/core/viewport';

// Auto-initialized on instantiation
const tabs = new YaiTabs();
const swype = new YaiTabsSwipe().setInstance(tabs).watchHooks();
üì¶ Install: npm install @yaijs/yeh @yaijs/core

Granular Imports (Tree-Shakeable) Optimal

Import only what you need. Perfect for custom builds and optimal bundle sizes. Note: When using granular imports, you'll need to install YEH separately. 

// YEH must be installed separately for granular imports
import { YEH } from '@yaijs/yeh';

// Import individual YaiJS components
import { YaiCore } from '@yaijs/core/core';
import { YaiTabs } from '@yaijs/core/tabs';
import { YaiTabsSwipe } from '@yaijs/core/swype';
import { YaiAutoSwitch } from '@yaijs/core/autoswitch';
import { YaiViewport } from '@yaijs/core/viewport';
import { YaiSearchAndClick } from '@yaijs/core/search-and-click';

// Components auto-initialize - only what you import gets bundled
const tabs = new YaiTabs();
‚ö° Best for tree-shaking - your bundler only includes what you use

YEH Only (Standalone) Foundation

Use YEH without YaiJS components. Perfect when you only need the event delegation system for your own custom components. 

// Install: npm install @yaijs/yeh
import { YEH } from '@yaijs/yeh';

// Use YEH for your own event delegation
const hub = new YEH();
hub.on('.my-button', 'click', (event) => {
    console.log('Button clicked!', event.target);
});
üì¶ YEH standalone: ~8 KB uncompressed, ~3 KB gzipped

üåê CDN Imports (Direct Browser Usage)

ESM Import from CDN Recommended

Import directly from JSDelivr CDN. Perfect for quick prototypes and no-build setups. 

// ESM import from CDN (auto-gzipped by JSDelivr)
import { YaiTabs, YaiTabsSwipe, YEH }
    from 'https://cdn.jsdelivr.net/npm/@yaijs/core/dist/yai-bundle.js';

// Or full bundle
import { YaiTabs, YaiAutoSwitch, YEH }
    from 'https://cdn.jsdelivr.net/npm/@yaijs/core/dist/yai-bundle-full.js';

// Components auto-initialize on instantiation
const tabs = new YaiTabs();
const autoSwitch = new YaiAutoSwitch().start();
üöÄ JSDelivr provides automatic Brotli/gzip compression and global CDN caching

Script Tag with window.YaiJS Global Browser

Use script tags to load YaiJS and access via window.YaiJS global. Works in any browser without build tools. 

<!-- Load bundle via script tag -->
<script type="module" src="https://cdn.jsdelivr.net/npm/@yaijs/core/dist/yai-bundle.js"></script>

<script type="module">
    // Access via window.YaiJS global - components auto-initialize
    const { YaiTabs, YaiTabsSwipe, YEH } = window.YaiJS;

    const tabs = new YaiTabs();
    const swype = new YaiTabsSwipe().setInstance(tabs).watchHooks();
</script>
üåç Works everywhere - no build tools required, all browsers supported

Direct File Access via GitHub CDN Advanced

Access individual files directly from GitHub via JSDelivr. Useful for custom setups and experimentation. 

// Direct file access from GitHub repo
import { YaiTabs }
    from 'https://cdn.jsdelivr.net/gh/yaijs/yai/tabs/yai-tabs.js';

import { YaiTabsSwipe }
    from 'https://cdn.jsdelivr.net/gh/yaijs/yai/utils/yai-tabs-swipe.js';

// Auto-initialized
const tabs = new YaiTabs();

üí° Pro Tips

  • ESM-only architecture: All files use ES modules - requires <script type="module"> in browsers
  • YEH + YaiJS relationship: YEH (@yaijs/yeh) is standalone, YaiJS builds on it
  • NPM users: Install both packages: npm install @yaijs/yeh @yaijs/core
  • CDN users: Use bundles (yai-bundle.js) - YEH is included automatically
  • Bundles are CDN-only: Bundle files only work via CDN, not via npm install
  • Tree-shaking: Use granular imports (@yaijs/core/tabs, etc.)
  • Just event delegation: Install only @yaijs/yeh if you don't need tabs
  • No build tools: Use CDN bundles with script tags (must be type="module")
  • TypeScript: All packages include full type definitions (.d.ts files)
  • Browser support: Chrome 61+, Firefox 60+, Safari 11+, Edge 79+ (ESM requirement)
